Detailed Description

This file contains methods and tasks needed to setup and run a web server hotspot from an ESP32 microcontroller. A lot of this code is taken from John Ridgely's in-class example which sets up a WiFi hotspot on an ESP32; the rest of it, including most of the HTML handling and live plotting, is generated by ChatGPT 5.1.

Callback function that responds to HTTP requests without a subpage name.

This creates a page which includes forms to command speed and torque, along with live gain tuning. It includes a live readout of the commanded speed and actual speed of the motor.

◆ handle_Speed()

void handle_Speed ( void )

HTTP handler which reports the current measured motor speed.

When a client accesses the /speed endpoint, this function reads the latest RPM value from the shared variable speed_actual and returns it as plain text. This endpoint is intended for JavaScript polling to generate real-time plots in the browser UI.
